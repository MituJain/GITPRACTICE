Here are the important Linux files/locations for networking troubleshooting:
DNS & Name Resolution
/etc/resolv.conf - DNS server configuration
/etc/hosts - Static hostname to IP mappings
/etc/nsswitch.conf - Name service switch config (resolution order)
/etc/hostname - System hostname
Network Configuration
/etc/network/interfaces - Network interfaces (Debian/Ubuntu)
/etc/sysconfig/network-scripts/ - Network configs (RHEL/CentOS)
/etc/netplan/*.yaml - Network config (Ubuntu 18+)
/etc/NetworkManager/ - NetworkManager configs
/etc/systemd/network/ - systemd-networkd configs
Important System Files
/proc/net/ - Real-time network stats
/proc/net/tcp - TCP connections
/proc/net/udp - UDP connections
/proc/net/route - Routing table
/sys/class/net/ - Network interface info
Firewall & Security
/etc/iptables/ - iptables rules
/etc/firewalld/ - firewalld config
/etc/ufw/ - UFW firewall config
/etc/hosts.allow - TCP wrapper allowed hosts
/etc/hosts.deny - TCP wrapper denied hosts
Logs (Critical for Debugging)
/var/log/syslog - System log (Debian/Ubuntu)
/var/log/messages - System log (RHEL/CentOS)
/var/log/kern.log - Kernel log
/var/log/dmesg - Boot messages
journalctl -u NetworkManager - NetworkManager logs
journalctl -u systemd-networkd - systemd-networkd logs
Service Configuration
/etc/ssh/sshd_config - SSH server config
/etc/ssh/ssh_config - SSH client config
/etc/services - Port to service name mappings
/etc/protocols - Protocol definitions
Useful Commands to Check These
cat /etc/resolv.conf - Check DNS servers
cat /etc/hosts - Check static mappings
ip route - Check routing table
systemctl status NetworkManager - Network service status
tail -f /var/log/syslog | grep -i network - Monitor network logs
